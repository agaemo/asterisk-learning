# Terraform とは何か・このプロジェクトで何をしているか

## Terraform とは

「インフラをコードで管理するツール」です。

### コードがなかった時代

AWSでサーバーを立てるとき、従来はこういう手順でした:

1. AWS コンソール（画面）を開く
2. VPC を手動で作成
3. サブネットを手動で作成
4. セキュリティグループを手動で作成
5. EC2 を手動で起動
6. Elastic IP を手動で割り当て
…

**問題点:**
- 同じ環境を再現しようとしても「どう設定したか」が記録に残らない
- 間違えたとき何を変更したか分からない
- 削除するときも手動で1つずつ消さないといけない

### Terraform を使うと

設定をコード（`.tf` ファイル）で書いておけば:

```bash
terraform apply   # コードの通りにAWSリソースを全部作ってくれる
terraform destroy # コードで作ったリソースを全部削除してくれる
```

1コマンドで完結します。「どう設定したか」がコードに残るので、再現・修正・削除が簡単です。

## このプロジェクトで作るAWSリソース

```
インターネット
     │
     ▼
┌─────────────────────────────────┐
│  VPC（仮想ネットワーク）         │
│  ┌───────────────────────────┐  │
│  │  パブリックサブネット      │  │
│  │  ┌─────────────────────┐  │  │
│  │  │  EC2（サーバー）     │  │  │
│  │  │  ・Asterisk が動く   │  │  │
│  │  │  ・Elastic IP 割当済 │  │  │
│  │  └─────────────────────┘  │  │
│  └───────────────────────────┘  │
│                                 │
│  セキュリティグループ            │
│  ・SSH(22): 自分のIPのみ許可    │
│  ・SIP(5060): 自分のIPのみ許可  │
│  ・RTP(10000-20000): 同上       │
└─────────────────────────────────┘
```

### 各リソースの役割

| リソース | 役割 | 身近な例え |
|----------|------|-----------|
| **VPC** | AWS上の仮想ネットワーク | マンションの棟 |
| **サブネット** | VPC内の区画 | マンションの部屋の区画 |
| **インターネットゲートウェイ** | VPCとインターネットの接続口 | マンションの玄関 |
| **ルートテーブル** | パケットの経路設定 | 「インターネット行きはゲートウェイへ」という案内板 |
| **セキュリティグループ** | 通信の許可/拒否ルール | ドアの鍵（特定のIPだけ入れる） |
| **EC2** | 仮想サーバー | レンタルサーバー |
| **Elastic IP** | 固定グローバルIPアドレス | 固定電話番号（IPアドレス版） |

## Terraform の基本コマンド

### `terraform init`（初期化）

```bash
terraform -chdir=terraform init
```

初めて使うときだけ実行します。プロバイダー（AWS を操作するためのプログラム）をダウンロードします。

### `terraform plan`（確認）

```bash
terraform -chdir=terraform plan
```

「apply を実行したら何が作られるか」をプレビューします。**実際には何も変わりません。**

```
+ aws_vpc.main will be created
+ aws_subnet.public will be created
+ aws_instance.asterisk will be created
...
```

`+` は新規作成、`~` は変更、`-` は削除を意味します。

### `terraform apply`（実行）

```bash
terraform -chdir=terraform apply
```

実際に AWS にリソースを作成します。実行前に plan の内容が表示され、`yes` と入力すると実行されます。

### `terraform destroy`（削除）

```bash
terraform -chdir=terraform destroy
```

Terraform で作成したリソースをすべて削除します。**Elastic IP は削除するまで課金されるため、学習終了後は必ず実行してください。**

## Terraform の状態管理（terraform.tfstate）

Terraform は「今 AWS に何を作ったか」を `terraform.tfstate` というファイルで管理しています。

```
terraform/
└── terraform.tfstate   ← "VPC: vpc-xxxxx を作った" などの記録
```

このファイルが壊れると Terraform が状態を把握できなくなるため:
- **git にコミットしない**（`.gitignore` で除外済み）
- **手動で編集しない**

## このプロジェクトのモジュール構成

Terraform のコードを3つのモジュール（部品）に分けています。

```
terraform/
├── main.tf          ← 3つのモジュールを呼び出す「司令塔」
├── variables.tf     ← 変数の定義（my_ip など）
├── outputs.tf       ← apply 後に表示する情報（Elastic IP など）
├── terraform.tfvars ← 変数の実際の値（gitignore対象）
└── modules/
    ├── vpc/         ← VPC・サブネット・ゲートウェイを作る部品
    ├── sg/          ← セキュリティグループを作る部品
    └── ec2/         ← EC2・Elastic IPを作る部品
```

モジュールに分けている理由: フェーズ2でTwilioを追加するとき、`sg/` モジュールだけ変更すれば済むようにするためです。

## SIP と RTP について（Asterisk の通信プロトコル）

電話の仕組みは「制御」と「音声」が別のプロトコルで動いています。

### SIP（ポート 5060）

「電話をかける・受ける・切る」という**制御信号**を扱います。

```
Zoiper（スマホ）             Asterisk
     │  "1002 に電話したい"       │
     │──────────────────────────>│  SIP（制御）
     │  "つながったよ"            │
     │<──────────────────────────│
```

### RTP（ポート 10000-20000）

実際の**音声データ**を送受信します。SIP で接続が確立した後、別の経路で音声が流れます。

```
Zoiper（スマホ）             Asterisk            Zoiper（PC）
     │                           │                    │
     │<══════════════════════════════════════════════>│  RTP（音声）
```

セキュリティグループでこの両方を開けているのはこのためです。

## よくある疑問

**Q: Elastic IP がないとダメですか？**

A: EC2 は起動するたびに IP アドレスが変わります。Zoiper の接続先 IP が変わると再設定が必要になるため、Elastic IP で固定しています。

**Q: t3.micro で性能は足りますか？**

A: 学習用の内線通話（数本）なら十分です。Asterisk は軽量で、t3.micro（2 vCPU・1 GB RAM）でも問題なく動きます。

**Q: User Data とは何ですか？**

A: EC2 が起動したときに**自動で実行されるスクリプト**です。このプロジェクトでは `scripts/install.sh` を User Data として渡しており、Asterisk のインストールが自動で行われます。ただし完了まで 10〜15 分かかります。
